## Interactivity

*This experiment explored how interaction can transform data from something static into something dynamic and engaging. Building on my Week 1 emotional data portrait, we used the p5.js web editor to experiment with interactive sketches that respond to user input and movemen*

## Process

*We began by experimenting with the p5.js web editor, using basic functions such as shapes, colour, and positioning to understand how visuals are constructed through code. This helped me build a foundation for creating more complex sketches.*

*I then explored interactivity by creating sketches that respond to user input. One of my main experiments was a kaleidoscope-style drawing tool, where lines are repeated through rotation and symmetry. This allowed me to create layered, evolving patterns that change depending on how the user moves their mouse.*

*I also experimented with randomness in colour and form, which meant that each interaction produced a slightly different outcome. This made the experience feel more dynamic and less controlled compared to my hand-drawn work.*

## Connection to week 1

*This experiment connects directly to my Week 1 data drawing, where I explored my emotions, interactions, and thoughts. In contrast to the fixed nature of the drawing, the interactive kaleidoscope represents how emotions are constantly changing and influenced by different situations.*

*The shifting and repeating patterns reflect how emotions can overlap, evolve, and respond to different inputs, rather than existing as static states.*

## Reflection 

*This experiment showed me that interactivity allows data to be experienced in a more immersive and expressive way. Unlike my hand-drawn data portrait, which presents a fixed moment, the interactive sketch allows for continuous change and variation.*

*The kaleidoscope was particularly effective in exploring how simple inputs can create complex visual outcomes. It made me think about how emotions and experiences are layered and interconnected, rather than isolated.*

*One challenge I faced was understanding how to structure the code to achieve the visual effects I wanted. However, through experimentation and using AI tools to support my learning, I was able to gradually build more complex interactions.*

*If I had more time, I would develop this further by linking specific interactions more directly to my emotional data, creating a clearer relationship between the data and the visual output.*

## AI Usage Statement

*I used ChatGPT as a tool to support my coding process in p5.js. I described the types of interactive sketches I wanted to create, such as a kaleidoscope-style drawing, and used the generated code as a starting point to experiment with different visual outcomes.*

*Rather than copying the code directly, I adjusted and tested it within the p5.js web editor to better understand how it worked. This allowed me to explore concepts such as symmetry, rotation, and randomness, which I then applied to my own sketches.*

*Using ChatGPT helped me experiment beyond my current coding knowledge and supported a trial-and-error approach to learning. However, I still needed to interpret and adapt the code to suit my ideas and ensure it aligned with my project.*
